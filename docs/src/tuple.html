    When called with any number of arguments, this function returns an object that inherits from tuple.prototype and is guaranteed to be === any other tuple object that has exactly the same items. In computer science jargon, tuple objects are “internalized” or just “interned,” which allows for constant-time equality checking, and makes it possible for tuple objects to be used as Map or WeakMap keys, or stored in a Set.

    export default function tuple(...items) {
  return intern(items);
}
